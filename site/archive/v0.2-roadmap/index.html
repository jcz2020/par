    P-A-R v0.2 — 论文 + 嵌入式引擎路线图¶

    Created: 2026-05-28 Status: Planning

    项目定位¶

    P-A-R 是第一个在类型安全运行时中同时实现全部四个维度的 Agent Runtime：

    维度 说明
    Tool Result ADT handler_result 5-case sum type，编译期穷尽匹配
    形式化状态机 8-state LTS，validate_transition 拒绝非法转换
    表达式 DSL 资源受限求值器（深度≤10，节点≤1000）
    可组合中间件 continuation-passing pipeline，5 hooks

    论文叙事：λ_A (arXiv:2604.11767) 证明了类型安全 agent 组合的理论可行性（Coq 模型），但没有实现。P-A-R 是该理论的第一个完整可运行实现，并且额外提供了形式化状态机和表达式 DSL。

    Phase 1：论文准备（2-3 周）¶

    目标¶

    投 ICFP 2026 Workshop 或 ML Workshop

    论文大纲¶

    "P-A-R: A Type-Safe Agent Runtime with Formal State Guarantees"

    • §1 Introduction: dynamic-typed agent frameworks 的运行时错误问题
    • §2 Background: λ_A, Shepherd (arXiv:2605.10913), CTEGs (arXiv:2604.17557)
    • §3 State Machine: 8-state LTS, validated transitions, 3 terminal states
    • §4 Expression DSL: big-step operational semantics, resource-bounded evaluation
    • §5 Middleware: continuation-passing composition, associativity proof/counterexample
    • §6 Evaluation: 111 tests + benchmark vs Python (100 tool-calling scenarios)
    • §7 Related Work (λ_A, oharness_tools, tower-llm, Shepherd, Semantic Kernel)
    • §8 Conclusion

    关键数据点¶

    • handler_result 5-case ADT vs Python any：构造 100 个工具调用场景，统计编译期捕获 vs 运行时暴露
    • validate_transition：构造 50 个非法状态转换，全部被类型系统拒绝
    • Expression evaluator：资源限制防止 DoS（深度≤10，节点≤1000）

    Phase 2：引擎化改造（3-4 周）¶

    2.1 工具接口与实现分离（最重要）¶

    现状：tool_binding.handler 是函数类型，不可序列化。

    (* 现在 *)
type tool_binding = {
  handler : Yojson.Safe.t -> cancellation_token -> handler_result;
  ...
}

(* 目标 *)
type tool_descriptor = {
  name : string;
  description : string;
  input_schema : Yojson.Safe.t;
  permission : tool_permission;
}

type tool_registry
val register : tool_registry -> descriptor:tool_descriptor ->
               handler:(Yojson.Safe.t -> cancellation_token -> handler_result) -> unit
val resolve : tool_registry -> string -> tool_binding option

    收益：agent_config 可序列化，工具实现运行时注入，为 FFI 铺路。

    2.2 Provider HTTP 代码去重¶

    现状：OpenAI 和 Anthropic provider 有 80% 重复代码（URL 解析、chunked encoding、TLS 配置）。

    目标：提取 lib/providers/http_client.ml 共享 HTTP 层。

    2.3 中间件状态隔离¶

    现状：retry.ml 用 ref 计数器，跨 agent 共享。

    目标：per-invocation 状态，并发安全。

    2.4 C FFI 层¶

    // par_ffi.h
ParRuntime* par_init(const char* config_json);
int        par_register_tool(ParRuntime* rt, const char* name,
                              ParToolHandler handler, void* user_data);
int        par_invoke(ParRuntime* rt, const char* agent_id,
                       const char* message, char** response);
void       par_shutdown(ParRuntime* rt);

    实现路径：OCaml Callback.register + Caml.startup。

    2.5 Workflow 持久化¶

    现状：Sub_workflow 和 Human_approval 是 stub（返回 error / true）。

    目标：workflow checkpoint 序列化，崩溃可恢复。

    优先级¶

    先做 Phase 2.1 + 2.2 + 论文 §3-5，同步推进： - 工具接口分离是引擎化前提 - HTTP 去重消除技术债 - 形式化不需要改代码，可纯写
