# v0.7.0–v0.7.2 Audit Findings — Fix List

## P0 — Must Fix (10 issues)

### 1. `Auto` search mode is dead code
- **File**: `lib/memory/sqlite_memory.ml:482-483`
- **Bug**: When `Auto` is passed explicitly, `resolve_mode` returns `Auto` unchanged, and `search` does `search_fts` — pure FTS-only. The `.mli` documents `Auto` as "Smart default: Hybrid if embedding available, else Keyword_only". The smart resolution only happens when `mode=None` (not passed).
- **Fix**: In `resolve_mode`, handle `Some Auto` the same as `None` — check embedding availability and resolve to `Hybrid` or `Keyword_only`.

### 2. `update()` never deletes the old row — data leak
- **File**: `lib/memory/sqlite_memory.ml:496-529`
- **Bug**: `update` INSERTs a new row with a new `ext_id` but never DELETEs the old one. After update: orphaned row persists in `memory_entries`, orphaned embedding persists in `memory_entries_vec`, stale FTS entry persists. Data leak grows with every update call.
- **Fix**: Before INSERT, `DELETE FROM memory_entries WHERE ext_id = ?` for the old entry. Or change to a true SQL UPDATE.

### 3. `invoke_structured` bypasses `Invoke_context` entirely
- **File**: `lib/core/runtime.ml:925-964`
- **Bug**: No `Invoke_context.create` or `with_context` wrapping. Consequences: (a) `system_prompt_appendix` not accepted as parameter, (b) `Invoke_context.get_current()` returns `None` inside `run_structured`, (c) no per-call metrics isolation, (d) no per-call hooks snapshot.
- **Fix**: Add `?system_prompt_appendix:string` parameter, create `Invoke_context.t`, wrap the call in `Invoke_context.with_context`.

### 4. `run_structured` doesn't inject `system_prompt_appendix`
- **File**: `lib/core/engine.ml:282-337`
- **Bug**: Compare `run_agent` at L554 which appends `Invoke_context.appendix_text()` to system prompt. `run_structured` constructs `sys_prompt` with NO appendix injection.
- **Fix**: Add `^ Invoke_context.appendix_text ()` to the system prompt construction in `run_structured`.

### 5. PDF loader throws `Failure` instead of returning typed error
- **File**: `lib/documents/pdf_loader.ml:94-96`
- **Bug**: `failwith` raises `Failure` exception inside the thunk. When called via `Directory_loader.load`, the exception propagates uncaught — crashes entire directory scan.
- **Fix**: Return `Error (Document.Extraction_failed (...))` instead of `failwith`. Wrap `thunk ()` calls in `Directory_loader` with exception handling.

### 6. HTML loader: no `Sys.file_exists` check
- **File**: `lib/documents/html_loader.ml:8`
- **Bug**: Unlike ALL other loaders (text:11, markdown:61, csv:11, pdf:85), HTML loader never checks file existence after workspace admission. Missing file → uncaught `Sys_error`.
- **Fix**: Add `Sys.file_exists` check before `open_in`, return `Error (File_not_found ...)`.

### 7. HTML loader: no exception safety — file handle leak
- **File**: `lib/documents/html_loader.ml:8-10`
- **Bug**: `close_in fh` is not in `Fun.protect ~finally`. If `really_input_string` or `Soup.parse` throws, FD leaks. Text and Markdown loaders correctly use `Fun.protect`.
- **Fix**: Wrap in `Fun.protect ~finally:(fun () -> close_in fh)`.

### 8. 5 FFI functions call `caml_copy_string`/`caml_copy_double` INSIDE lock
- **File**: `lib/ffi/par_ffi.c:364,380,419-421,641,653`
- **Bug**: File's own comment (L3-5) says "Allocation done OUTSIDE the lock to prevent longjmp from OOM skipping unlock". Violations: `par_embed` (L364), `par_add_documents` (L380), `par_invoke_with_rag` (L419-421), `par_set_request_timeout` (L641), `par_set_vec_extension_path` (L653).
- **Fix**: Move all `caml_copy_string`/`caml_copy_double` calls to BEFORE `PAR_MUTEX_LOCK`.

### 9. `vec_extension_path` override mechanism broken
- **File**: `lib/ffi/par_capi.ml:1164-1165`
- **Bug**: `vec_extension_path` is a `let` binding (evaluated once at module init). `set_vec_extension_path_override` sets a `ref` AFTER init, but `vec_extension_path` already read it. Python binding's override NEVER takes effect.
- **Fix**: Change `vec_extension_path` from `let` binding to a function `let vec_extension_path () = ...` that reads the ref each time. Update all call sites.

### 10. `sqlite_memory.ml` missing Windows vec0 vendor path
- **File**: `lib/memory/sqlite_memory.ml:34-43`
- **Bug**: `resolve_vec_extension_path()` candidates include `vendor/sqlite-vec/linux-x86_64/` and `macos-aarch64/` but NOT `windows-x86_64/`. On Windows, `vec0.dll` is never found. Compare with `par_capi.ml:1188` which correctly includes it.
- **Fix**: Add `Filename.concat cwd ("vendor/sqlite-vec/windows-x86_64/" ^ so_name)` to candidates.

## P1 — Should Fix (6 issues)

### 11. Double-appendix on conversation resume
- **File**: `lib/core/engine.ml:1027-1036`
- **Bug**: When resuming a conversation, the existing system message (already baked with appendix from prior invoke) gets CURRENT invoke's appendix appended again. Multiple resumes → unbounded system message growth.
- **Fix**: Strip any existing appendix from the system message before appending new one. Or store "bare" system prompt separately.

### 12. `LOADER` module type is dead code
- **File**: `lib/documents/document.ml:62-65`
- **Bug**: `module type LOADER` defined with `lazy_load : unit -> Seq.t` but none of 5 loaders implement it. `Seq.t` streaming promise completely unfulfilled.
- **Fix**: Either implement LOADER in each loader (add `lazy_load`), or remove the dead module type and document the `make` pattern as canonical.

### 13. CSV loader: `Csv.load` can throw uncaught
- **File**: `lib/documents/csv_loader.ml:17`
- **Bug**: `Csv.load full_path` not wrapped in try/with. Malformed CSV → uncaught exception through Directory_loader.
- **Fix**: Wrap in `try ... with exn -> return Error (Extraction_failed (...))`.

### 14. `Directory_loader.load` default map is empty — silent no-op
- **File**: `lib/documents/directory_loader.ml:13`
- **Bug**: `?(map : extension_map = [])` — calling without `~map` loads ZERO files.
- **Fix**: Change default to `Directory_loader.default_map` instead of `[]`.

### 15. No embedding dimension validation
- **File**: `lib/memory/sqlite_memory.ml:155-171`
- **Bug**: `insert_embedding` calls `float_array_to_blob vec` without checking `Array.length vec = dimension`. Wrong-dimension vector silently corrupts vec0 index. `vector_store.ml:95-100` has this check.
- **Fix**: Add `if Array.length vec <> t.dimension then () (* skip or error *)` before blob conversion.

### 16. `invoke_handle_cancel` TOCTOU race
- **File**: `lib/core/invoke_context.ml:69-71`
- **Bug**: Sets `Atomic.set h.status Cancelled` BEFORE `Cancellation.request_cancel`. If fiber completes between these two ops, status flips Cancelled→Completed.
- **Fix**: Use `Atomic.compare_and_set` or set status AFTER cancel is observed.
