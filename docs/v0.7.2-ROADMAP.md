## 1. Background

v0.7.1 shipped concurrency architecture and memory module but deferred Windows support (#5) and vector-based semantic recall. v0.7.2 addresses both gaps:

- **Windows (#5)**: Eio.Process is a `failwith` stub on Windows. Solution: capability registry (R1-architecturally-correct) gates process spawning behind runtime detection. Core runtime (agents, LLM, persistence, memory, workflows) works on Windows. Bash tool and MCP stdio return typed errors.
- **Vector Memory**: vec0 virtual table alongside FTS5, with Reciprocal Rank Fusion (RRF) hybrid search. Smart default: Hybrid when embedding configured, Keyword_only otherwise.
- **SDK Docs**: 7 of 12 v0.7.1 APIs were undocumented. All gaps filled with EN+ZH docs.
