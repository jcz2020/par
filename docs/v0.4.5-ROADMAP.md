# v0.4.5 Roadmap — CLI Bugfix Beta

> **Status**: Active. Created 2026-06-18.
> **Theme**: Fix all 11 CLI bugs found during real user testing of v0.4.4-beta.
> **Type**: Bugfix-only beta. No new features. Patch bump from v0.4.4-beta.
> **Predecessor**: [v0.4-ROADMAP.md](v0.4-ROADMAP.md) (Phase 5 = v0.4.4).

---

## Rationale

v0.4.4-beta.20260618 shipped with 933 OCaml tests passing, but real-world CLI usage on macOS surfaced 11 bugs ranging from P0 (CLI unusable) to P2 (cosmetic). This roadmap fixes all 11 before any new feature work.

v0.4.4 stable is **skipped** — the beta tag already serves as the release artifact. v0.4.5-beta will be the next tag.

---

## Bug Inventory (11 total)

### P0 — Blocking (4)

| ID | Symptom | Root Cause |
|----|---------|------------|
| PAR-zlm | `par ask` after tool call → LLM API 400 "tool call id is invalid (2013)" | engine.ml:266 overwrites LLM's tool_call.id; anthropic streaming emits index not real id |
| PAR-rev | Tool succeeds but no LLM follow-up output | Same as PAR-zlm (id mismatch causes silent LLM failure) |
| PAR-xmb | REPL dies silently after tool failure | engine.ml:89 no try/with; main.ml:500 only catches End_of_file |
| PAR-mhs | REPL causes terminal crash | Bare `input_line stdin` passes through binary/control chars |

### P1 — Important (5)

| ID | Symptom | Root Cause |
|----|---------|------------|
| PAR-pso | Conversation context missing assistant replies | Same root cause as PAR-zlm (broken tool_call_id pair) |
| PAR-ngb | LLM events not persisted to session | Llm_request_sent / Llm_response_received never emitted by engine |
| PAR-v5a | `par ask 本地有哪些文件夹？` fails "too many arguments" | Cmdliner single positional string rejects multi-token input |
| PAR-wmj | Arrow keys produce escape sequence garbage | Bare stdin has no readline/ANSI support |
| PAR-r05 | Chinese characters can't be fully deleted | Bare stdin has no UTF-8 awareness |

### P2 — Cosmetic (2)

| ID | Symptom | Root Cause |
|----|---------|------------|
| PAR-0yx | Tool result content not in events | Tool_completed event lacks result_preview field |
| PAR-br3 | `/health` outputs raw JSON, not human-readable | main.ml:466 calls print_json directly |

## Work Units (6)

Grouped by root cause for combined fixes.

### WU-1 · Tool_call_id Preservation (PAR-zlm + PAR-rev + PAR-pso)

**Files**: `lib/core/engine.ml:266`, `lib/providers/anthropic_provider.ml:234`, `lib/providers/openai_provider.ml:213-218`

Stop overwriting LLM-returned tool_call.id with fresh Task_id. Thread event_task_id separately for event correlation. Fix Anthropic streaming to emit real tc_id (not array index). Fix OpenAI streaming to read real `id` field from chunks.

### WU-2 · REPL Exception Safety (PAR-xmb)

**Files**: `lib/core/engine.ml:89`, `bin/main.ml:442-500`

Wrap tool handler call in try/with; convert exceptions to `Error { category = Internal _; ... }`. Add defense-in-depth exn catch in REPL loop (continue instead of crash). Reuse existing `Internal` error category (no new variant).

### WU-3 · LLM Event Emission (PAR-ngb)

**Files**: `lib/core/engine.ml:236`, `lib/core/context_manager.ml:104`

Emit `Llm_request_sent` before and `Llm_response_received` after every real LLM call. Currently these event variants are defined, formatted, and counted in stats — but never actually fired.

### WU-4 · CLI Multi-Token Arg Parsing (PAR-v5a)

**Files**: `bin/main.ml:72-75`, `bin/main.ml:547-572`, new `test/test_cli_args.ml`

Switch from `pos 0 (some string)` to `pos_left` or equivalent to consume all remaining argv tokens. Join with spaces before passing to Runtime.invoke.

### WU-5 · REPL Readline Migration (PAR-wmj + PAR-r05 + PAR-mhs)

**Files**: `dune-project`, new `bin/repl.ml`, `bin/main.ml:442-500`, new `test/test_repl.ml`

Replace bare `input_line stdin` with `linenoise` (BSD-3, already installed locally). Adds UTF-8 awareness, arrow key support, and command history. Ctrl-C behavior: first press aborts current line, second press within 3 seconds exits REPL.

### WU-6 · Event Content + Health Format (PAR-0yx + PAR-br3)

**Files**: `lib/core/types.ml:398`, `lib/core/types.mli`, `lib/core/engine.ml:318,326`, `lib/persistence/persistence_common.ml`, `bin/main.ml:371,466,914`

Add `result_preview : string option` to `Tool_completed` event (truncated to 500 chars). Replace raw JSON `/health` output with colored human-readable format (runtime status, persistence status, last LLM call summary). No JSON fallback.
