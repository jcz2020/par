## Track B: RAG Foundation (the v0.5.0 actual content)

**Source issue**: PAR-ngh (P0)
**Strategic alignment**: STRATEGY.md §4 priority #3 (multi-provider — embeddings are LLM API surface).
**Why now**: LangChain's biggest use case. PAR's RAG gap is the single biggest "missing vs LangChain" item.

### Phase B.1 — Embedding API (Day 1-4)

- New `Runtime.embed` function: `message:string -> (float array, error_category) result`
- Provider implementations:
  - `openai_provider`: `POST /v1/embeddings` with `text-embedding-3-small/large`
  - `anthropic_provider`: Anthropic has no embeddings API — route through OpenAI compat (document explicitly)
  - `ollama_provider`: local embeddings via `nomic-embed-text`
  - `mock_provider`: deterministic fake vectors for tests

### Phase B.2 — Vector storage interface (Day 5-7)

- New `lib/core/vector_store.ml` + `.mli`
- Interface: `add : documents -> unit`, `search : query_vector -> k:int -> result list`, `delete : ids -> unit`
- Backend 1: SQLite + `sqlite-vec` extension (zero-dependency, ships with v0.5)
- Backend 2 (optional): External Qdrant/Milvus via HTTP (defer to v0.5.1 if scope balloons)

### Phase B.3 — Document chunking (Day 8-9)

- `lib/core/chunking.ml`:
  - `chunk_by_chars : text -> max_size:int -> overlap:int -> chunk list`
  - `chunk_by_tokens : text -> max_tokens:int -> overlap:int -> chunk list` (uses provider tokenizer if available)
  - `chunk_recursive : text -> separators:string list -> chunk list` (LangChain's RecursiveCharacterTextSplitter)

### Phase B.4 — RAG orchestration (Day 10-11)

- `Runtime.invoke_with_rag`:
  1. Chunk user query → embed
  2. Retrieve top-k from vector store
  3. Build augmented system prompt with retrieved context
  4. Call existing `Runtime.invoke`
- New event types: `Embedding_request_sent`, `Embedding_response_received`, `Retrieval_completed`

### Phase B.5 — Python FFI (Day 12)

- `Runtime.embed(message)` Python method
- `Runtime.add_documents(documents, metadatas)` Python method
- 5+ pytest tests

### Phase B.6 — Documentation (Day 13)

- New `docs/sdk/rag.md`: embeddings API, vector store, chunking, retrieval patterns
- How-to: "Build a basic RAG pipeline" (3 examples: short text, long doc, multi-doc)

**GATE**: User can build a working Q&A bot over a 100-page PDF using only `par-runtime` from PyPI (after v0.4.11 unblocks PyPI).
