# v0.5 Roadmap

> **Goal**: Ship platform-true wheels for ARM64 Linux and ARM64 macOS, in addition to the existing x86_64 Linux manylinux_2_28 wheel.
> **Rationale**: Current `pip install par-runtime` crashes on Apple Silicon Macs and ARM Linux servers (loads x86_64 ELF via ctypes). v0.5.0 fixes that without forcing source-build for non-x86_64 users.
> **Released**: 2026-06-21

### Why these three, not universal2

User explicit decision (2026-06-21): **`macos-13` (Intel) was permanently abandoned 2026-06-19** (`ci.yml` line 16). Intel Mac runners queue 24h+ then hit max-execution-time on the free tier. True `universal2` (which requires both `macos-13` + `macos-15` slices) is **NOT achievable** without paid GitHub Actions minutes.

Trade-off accepted:
- Apple Silicon Macs (the 2026 market) get a native wheel
- Intel Macs (<5% of PyPI 2026 traffic) fall back to source distribution (`pip install par-runtime` will compile from source on Intel Macs ‚Äî same as pre-v0.5.0 experience for that segment)
- Defer Intel Mac native wheel to v0.6+ (when/if user revisits the macos-13 decision or moves to a paid tier)

### Changes from v0.4.13

**`pypi-publish.yml`** ‚Äî refactored from single hardcoded job into a 3-job matrix:
- Job `build-wheel (linux-x86_64)` ‚Äî preserves v0.4.13 behavior (manylinux_2_28_x86_64 container + auditwheel)
- Job `build-wheel (linux-aarch64)` ‚Äî NEW: uses `quay.io/pypa/manylinux_2_28_aarch64:latest` container
- Job `build-wheel (macos-arm64)` ‚Äî NEW: `macos-15` runner, `delocate` instead of `auditwheel`, `macosx_11_0_arm64` tag
- `pypi-upload` job downloads all 3 artifacts and publishes them in one PyPI upload call

**`release-acceptance.yml`**:
- Removed `WHEEL_COUNT -ne 1` guard (lines 73-77 of v0.4.13) ‚Äî release now ships 3 wheels
- Added `linux-arm64` job: `debian:bookworm-arm64` container downloads the aarch64 wheel, runs `scripts/release-acceptance-test.py`
- Added `macos-arm64` job: `macos-15` runner downloads the arm64 wheel, runs the acceptance script via `pip install --user`
- `linux-x64` matrix remains: `debian:12`, `ubuntu:22.04`, `ubuntu:24.04` (3 containers √ó 1 wheel each)

**`docs/rules/release.md` line 178** ‚Äî "manylinux is v0.5+ stretch scope" ‚Üí "manylinux shipped in v0.4.13; ARM64 + macOS arm64 added in v0.5.0"
