# Release Process

## Prerequisites

```bash
opam install dune-release
```

## Release Steps

1. **Update version**
   - Edit `(version "...")` in `dune-project`
   - Add new entry to `CHANGES.md`

2. **Regenerate opam files**
   ```bash
   dune gen-opam-files
   git add *.opam CHANGES.md dune-project
   git commit -m "chore: bump version to vX.Y.Z"
   ```

3. **Tag the release**
   ```bash
   git tag -a vX.Y.Z -m "Release vX.Y.Z"
   git push origin main vX.Y.Z
   ```

4. **Create distribution**
   ```bash
   dune-release distrib
   ```

5. **Publish to GitHub**
   ```bash
   dune-release publish
   ```

6. **Submit to opam-repository**
   ```bash
   dune-release opam submit
   ```

7. **Monitor CI**
   - Watch opam-repository CI for build status across platforms
   - Fix any lint errors or build failures
   - Respond to reviewer feedback

## Notes

- `.opam` files are auto-generated by dune — edit `dune-project`, not `.opam` files
- `dune-release` creates one tarball for both packages (par + par_cli)
- opam-repository CI tests across: Linux, macOS, FreeBSD, multiple OCaml versions
- PostgreSQL is a hard dependency in v0.1.0 — future versions may make it optional
